# 2-structure.md (Структурирование проекта)

Этот воркфлоу используется, когда первый экран готов, чтобы «узаконить» архитектуру кода.

## Основные инструкции

1. **Разделение на логические блоки**:
   - Разделить код `index.html` на четкие семантические зоны:
     - `<header>`
     - `<main>`
     - `<footer>`

2. **Интеграция Hero-секции**:
   - Поместить созданную в рамках `1-start` Hero-секцию внутрь тега `<main>`.

3. **Глобальные стили**:
   - Определить общую ширину контейнера (например, `max-width: 1200px` или `1440px`).
   - Вынести глобальные стили (сброс, шрифты, цвета) в начало CSS файла или в `:root`.
   - Обеспечить наследование дизайна для всех будущих секций.

4. **Цель**:
   - Сделать верстку «сильной» и системной.
   - Предотвратить хаос («кашу») в коде при добавлении новых блоков нейросетью.
