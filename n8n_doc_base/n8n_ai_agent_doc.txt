        1. Базовые понятия n8n
        n8n — платформа автоматизации (workflow automation), где вы соединяете ноды
        (nodes) в граф, запускаемый триггерами и/или вручную. Нода принимает items (массив
        объектов), делает обработку и отдаёт items дальше по веткам.
        • Workflow — граф из нод и соединений.
        • Node — операция/интеграция (HTTP запрос, чтение из Google Sheets, отправка
        сообщения в Telegram и т.д.).
        • Credential — сохранённые секреты/ключи (API key, OAuth2 и т.д.), к которым
        привязываются ноды.
        • Execution — конкретный запуск workflow (с логами и данными).
        • Expressions — шаблоны в стиле {{ ... }}, чтобы подставлять данные из потока в
        параметры нод.
        Важно: один и тот же workflow можно собирать визуально в UI и/или программно через
        JSON-описание, которое экспортируется/импортируется.
        2. Workflow JSON: структура и правила
        Экспорт workflow в n8n — это JSON с ключевыми секциями nodes и connections. Ниже —
        практическая «спецификация» того, что ИИ агенту чаще всего нужно знать для
        генерации корректного workflow JSON.
        2.1. Минимальный пример workflow JSON
            {
              "name": "Demo: HTTP → Set",
              "nodes": [
                {
                  "id": "1",
                  "name": "HTTP Request",
                  "type": "n8n-nodes-base.httpRequest",
                  "typeVersion": 4,
                  "position": [420, 300],
                  "parameters": {
                    "url": "https://example.com/api",
                    "method": "GET",
                    "responseFormat": "json"
                  }
                },
                {
                  "id": "2",
                  "name": "Set",
                  "type": "n8n-nodes-base.set",
                  "typeVersion": 3,
                  "position": [700, 300],
                  "parameters": {
                    "keepOnlySet": true,
                    "values": {
                      "string": [
                        { "name": "source", "value": "={{ $node['HTTP Request'].json.url || 'example' }}" }
                      ]
                    }
                  }
                }
              ],
              "connections": {

                "HTTP Request": {

        Критично: объект connections ссылается на name нод (не на id). Поэтому имена должны
        быть уникальными внутри workflow.
        2.2. Поля объекта node
        • id: строка-идентификатор (часто UUID, но при импорте может быть и простое
        значение).
        • name: отображаемое имя ноды. Используется в connections.
        • type: тип ноды, например n8n-nodes-base.httpRequest.
        • typeVersion: версия описания ноды (важно для совместимости).
        • position: [x, y] координаты на канвасе.
        • parameters: параметры ноды (то, что вы заполняете в UI).
        • credentials (опц.): какие креды использует нода, например { "telegramApi": { "id":
        "...", "name": "..." } }.
        • disabled, continueOnFail, alwaysOutputData и др. (опц.): поведение выполнения.
        2.3. Поля connections: как описываются связи
        Связи описываются от выходов одной ноды к входам другой. В большинстве нод
        основной выход называется main и имеет массив выходных «портов».
            "connections": {
              "If": {
                "main": [
                  [ { "node": "Send True", "type": "main", "index": 0 } ],
                  [ { "node": "Send False", "type": "main", "index": 0 } ]
                ]
              }
            }
        Пример выше: у ноды If два выхода (ветки True/False). Каждый выход — список
        подключений.
        2.4. Expressions и доступ к данным
        Expressions в n8n пишутся внутри ={{ ... }} и позволяют обращаться к данным
        текущего item и к результатам других нод.
            ={{ $json.email }}
            ={{ $node["HTTP Request"].json.data.id }}
            ={{ $items("Some Node")[0].json }}
        Для генерации workflow JSON ИИ агенту важно: выражения — это строки, а не
        отдельный тип. Они живут внутри параметров нод.

        4. AI Agent (Tools Agent) в n8n
        AI Agent — root-нода из AI/LLM «кластерных» нод. Она связывается с Chat Model,
        Memory и инструментами (Tools), чтобы агент мог вызывать действия.
        4.1. Что изменилось в новых версиях
        В документации n8n отмечено, что до версии 1.82.0 в AI Agent можно было выбирать
        разные типы агента, но это убрали: теперь все AI Agent ноды работают как Tools Agent
        (рекомендованный и самый используемый режим).
        Также указано, что к AI Agent нужно подключить как минимум один tool (sub-node).
            Ключевая мысль:
            - современный AI Agent = Tools Agent
            - без tool-подключений агент работать не будет (нужно минимум 1 tool)
        4.2. Как собрать AI Agent workflow по «каноническому»
        паттерну
        • Chat Model: отдельная sub-нода (например OpenAI Chat Model / Anthropic Chat Model /
        Ollama Chat Model).
        • Memory: Simple Memory / Redis Chat Memory / Postgres Chat Memory и т.п.
        (опционально, но почти всегда нужно).
        • Tools: один или несколько инструментов (например: Calculator, Custom Code Tool, Call
        n8n Workflow Tool, HTTP Request Tool и др.).
        • Root: AI Agent нода, к которой сходятся Chat Model + Tools (+ Memory).
        Рекомендация для JSON-генерации: сначала сгенерируйте skeleton (root + sub-nodes +
        connections), затем заполняйте параметры моделей/инструментов.
        5. Как автоматически получить «все данные
        по нодам» (схемы параметров)
        Полный список нод — это только первый шаг. Чтобы ИИ агент «понимал, как строить
        ноды», ему нужна схема параметров (NodeDescription) каждой ноды. Это реально
        получить автоматически из npm-пакетов n8n.
        5.1. Два уровня данных
        • Каталог типов: имя ноды (node_key), строка type, путь к файлу реализации. Это
        лежит в dist/known/nodes.json.
        • Схема параметров: содержимое node.description (properties, inputs/outputs, defaults,
        etc.). Это находится внутри JS-класса ноды.

        5.2. Пример извлечения схемы ноды через Node.js
            // 1) Установите пакет с нодами
            //    npm i n8n-nodes-base
            // 2) Импортируйте класс ноды и выведите её описание (schema)
            const { HttpRequest } = require('n8n-nodes-base/dist/nodes/HttpRequest/HttpRequest.node');
            const node = new HttpRequest();
            console.log(node.description); // <-- здесь вся схема параметров (NodeDescription)
        5.3. Массовая выгрузка схем всех нод
            /**
             * Идея:
             * 1) прочитать dist/known/nodes.json
             * 2) для каждого node_key взять sourcePath
             * 3) require() по sourcePath, создать экземпляр, взять .description
             * 4) сохранить в единый JSON (node_key -> description)
             */
            const fs = require('fs');
            const known = JSON.parse(fs.readFileSync('node_modules/n8n-nodes-base/dist/known/nodes.json', 'utf8'));
            const out = {};
            for (const [nodeKey, meta] of Object.entries(known)) {
              const mod = require('n8n-nodes-base/' + meta.sourcePath);
              const Cls = mod[meta.className];
              const inst = new Cls();
              out[nodeKey] = inst.description;
            }
            fs.writeFileSync('ALL_NODE_DESCRIPTIONS.json', JSON.stringify(out, null, 2));
        Результат ALL_NODE_DESCRIPTIONS.json — это практически «полная база знаний по
        нодам», которую удобно «скормить» вашему ИИ агенту.
        6. Источники и как обновлять снимок
        Этот документ собран на основе официальной документации n8n и содержимого
        опубликованных npm-пакетов (UNPKG). Чтобы обновить снимок:
        • Откройте в UNPKG нужную версию n8n-nodes-base и скачайте dist/known/nodes.json
        и dist/known/credentials.json.
        • Пересоберите таблицы (каталог нод/кредов) и при необходимости выгрузите
        .description всех нод по скрипту выше.
        • Для AI/LLM нод используйте пакет @n8n/n8n-nodes-langchain (аналогичный подход).
